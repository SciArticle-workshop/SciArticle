# API Reference - Документация HTTP-взаимодействия между сервисами
Этот документ описывает HTTP-запросы, которыми обмениваются два сервиса - боты (`@SciArticleBot` и `@SciSourceBot`), для корректной передачи данных при взаимодействии друг с другом.

## Аутентификация
Все запросы защищены с помощью API токена, передаваемого в заголовке:

```
X-API-Token: <секретный_токен>
```
Токен хранится в .env и проверяется на стороне получателя.

## 1

### Отправка информации о запросах на статью/статьи, которые нужно удалить.


**URL:** `/api/request-pdf-expired/`

**Метод:** `POST`

**Описание:** Бот `@SciArticleBot` уведомляет бота `@SciSourceBot` о том, что запрос нужно удалить. Возможная причина: что срок действия запроса/запросов истёк или опубликован pdf-файл для проверки в общем чате `SciArticle Search`  

**Передаваемые параметры:**

| Поле              | Тип   | Описание                  |
|-------------------|-------|---------------------------|
| `message_id`      | int   | ID сообщения пользователя в чате с @SciSourceBot             |
| `chat_id`         | int   | Telegram ID пользователя  |
| `message_search_id`| int   | ID сообщения в общем чате SciArticle Search |

**Ответ:**

- `204 No Content` — успешно обработано  
- `400 Bad Request` — ошибка в запросе 
- `500 Internal Server Error` - ошибка сервера

## 2

### Повторная публикация запроса на статью

**URL:** `/api/new_request-pdf/`

**Метод:** `POST`

**Описание:** Бот `@SciArticleBot` отправляет боту `@SciSourceBot` запрос c данными для новой публикации запроса в общем чате, получает ответ с данными повторного запроса.

**Передаваемые параметры:**

| Поле     | Тип   | Описание      |
|----------|-------|---------------|
| `doi`    | str   | DOI статьи    |

**Ответ:**

- `200 OK` с JSON: `{ 'message_id': <int> }`— ID отправленного сообщения в общий чат SciArticle Search
- `400 Bad Request` — ошибка в запросе
- `500 Internal Server Error` - ошибка сервера

<br>

## 3

### Проверка наличия пользователя


**URL:** `/api/tg_user/<telegram_id>`

**Метод:** `GET`

**Описание:** Бот `@SciArticleBot` отправляет боту `@SciSourceBot` запрос c данными, чтобы тот проверил подписан ли пользователь на него.

**Ответ:**
- `200 OK` — пользователь найден
- `400 Bad Request` — ошибка в запросе  
- `404 Not Found` - пользователь не найден
- `500 Internal Server Error` - ошибка сервера

<br>

## 4

### Отправка информации о счетчике 

**URL:** `/api/user_counters/`

**Метод:** `POST`

**Описание:** Бот `@SciArticleBot` отправляет боту `@SciSourceBot` информацию про загрузки и проверки пользоватей, которые есть в `SciSourceBot`, получает ответ с данными.

**Передаваемые параметры:**

| Поле        | Тип    | Описание                      |
|-------------|--------|-------------------------------|
| `count`     | int    | Количество загрузок/проверок пользователя |
| `chat_id`   | int    | Telegram ID пользователя      |
| `count_type`| str    | `'upload'` или `'validation'` |
| `limit`     | int    | Порог для подписки            |

**Ответ:**

- `200 OK` с JSON: `{'chat_id': <int>, 'message_id': <int>}`— Telegram ID пользователя, ID отправленного сообщения пользователю
- `400 Bad Request` — ошибка в запросе 
- `500 Internal Server Error` - ошибка сервера

<br>

## 5

### Отправка pdf файла и данных

**URL:** `/api/upload-pdf/`

**Метод:** `POST`

**Описание:** Бот `@SciArticleBot` отправляет боту `@SciSourceBot` проверенный pdf файл (валидный) и данные.

**Передаваемые параметры:**

| Поле        | Тип            | Описание         |
|-------------|----------------|------------------|
| `file`      | файл (PDF)     | Файл документа (валидный) |
| `chat_id`   | int            | Telegram ID пользователя, загрузившего pdf          |
| `message_id`| int            | ID сообщения     |
| `doi`       | str            | DOI статьи       |

**Ответ:**

- `200 OK` — файл и данные получены  
- `400 Bad Request` — ошибка в запросе 
- `500 Internal Server Error` - ошибка сервера
<br>

## 6

### Отправка данных о истекших благодарственных сообщениях

**URL:** `/api/thank_message_delete/`

**Метод:** `POST`

**Описание:** Бот `@SciArticleBot` отправляет боту `@SciSourceBot` запрос, содержащий информацию о сообщении/сообщениях - пришло время удалить.

**Передаваемые параметры:**

| Поле         | Тип   | Описание                |
|--------------|-------|-------------------------|
| `message_id` | int   | ID сообщения в чате     |
| `chat_id`    | int   | Telegram ID пользователя|

**Ответ:**

- `204 No Content` — успешно  
- `400 Bad Request` — ошибка в запросе 
- `500 Internal Server Error` - ошибка сервера

## 7

### Выдача подписки пользователю

**URL:** `/api/grant-subscription/`

**Метод:** `POST`

**Описание:** Бот `@SciArticleBot` отправляет боту `@SciSourceBot` запрос на выдачу подписки, получает ответ с данными о подписке пользователя.

**Передаваемые параметры:**

| Поле         | Тип     | Описание                      |
|--------------|---------|-------------------------------|
| `telegram_id`| int     | Telegram ID пользователя      |
| `reason`     | str     | Причина: `'uploads'` или `'validations'` |
| `amount`     | int     | Количество месяцев подписки   |

**Ответ:**

- `200 OK` — JSON: `{'user_id': <int>, 'start_at': <str>, 'end_at': <str>}` - Telegram ID пользователя, дата начала подписки, дата окончания подписки  
- `400 Bad Request` — ошибка в запросе
- `404 Not Found` - пользователь не найден
- `500 Internal Server Error` - ошибка сервера
